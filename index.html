How I Work

From raw data to
clear decision.

The workflow I follow on every project — consistent, documented, and stakeholder-friendly at every step.

01

Understand the Business Question

Before touching data, I sit with the stakeholder to understand what decision this analysis needs to enable. The question shapes everything that follows — the data sources, the metrics, the format of the output.

02

Explore & Profile the Data

EDA in Python: distributions, nulls, outliers, and the weird stuff that always shows up in production data. I document findings so everyone can see the data's true shape before any model is built.

03

Build the Pipeline

SQL transformations, dbt models, or Python scripts — whatever keeps the data flowing cleanly. I write it for the person who has to maintain it in six months, which is usually me.

04

Analyse & Model

Statistical testing, segmentation, forecasting, or ML — calibrated to what the question actually needs. No XGBoost for questions that only need a pivot table.

05

Visualise & Present

Dashboards designed for the audience — executives need one number and a trend; analysts need drill-downs. Every chart has a point. Every slide answers a question.

06

Track Impact

I come back after the decision is made to measure whether the analysis was right. It closes the loop and makes the next analysis better.

Introducing NovaDB

The Universal Human-Readable Database Language — built for the future of AI, simplicity, and global scale.

MySQL · PostgreSQL · Oracle · MongoDB · Redis · Cassandra → NovaDB
📝

Plain English Queries

Write database operations the way you'd speak them. No cryptic syntax, no memorizing arcane commands — just natural language that works.

🧠

AI-Driven Type Inference

No INT, VARCHAR, or TEXT declarations. NovaDB reads your intent and infers the right data type automatically — powered by intelligent schema analysis.

🔗

Auto-Relationships

Foreign keys are a thing of the past. NovaDB detects and manages relationships between your data automatically. Just describe what you need.

⚡

Massive Scalability

From a solo side project to a billion-row production database — NovaDB scales horizontally with zero config changes to your queries.

🌐

Universal Data Models

Relational, document, graph, key-value, real-time streaming — one language rules them all. Switch data models without rewriting your queries.
